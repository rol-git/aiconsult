# Настройка системы поддержки

## Быстрая установка

### 1. Добавьте поле sender_id в БД
```bash
cd server
psql -d your_database -f add_sender_id.sql
```
Или автоматически (Windows):
```cmd
cd server
fix_sender_id.bat
```

### 2. Создайте оператора
```bash
cd server
python create_support_user.py
```

### 3. Запустите систему
```bash
# Терминал 1
cd server
python app.py

# Терминал 2
cd client
npm start
```

## Использование

### Пользователь:
1. Отправьте любое сообщение боту
2. После ответа появится кнопка "👤 Позвать оператора"
3. Нажмите на неё - тикет создастся только если есть **онлайн операторы**

### Оператор:
1. Войдите с email/паролем созданного оператора
2. **Важно**: оператор считается онлайн только когда подключен через WebSocket
3. Увидите панель поддержки с тикетами
4. Кликните на тикет - он автоматически назначится вам
5. Отвечайте пользователю в реальном времени

## Важно: Онлайн операторы

**Тикеты создаются только для онлайн операторов!**
- Оператор должен быть залогинен
- Должна быть открыта панель поддержки
- WebSocket должен быть подключен

Если операторов нет онлайн - пользователь увидит сообщение о горячей линии.

## Проверка системы
```bash
cd server
psql -d your_database -f check_system.sql
```

## Особенности работы:
- ✅ **WebSocket в реальном времени**:
  - Оператор и пользователь общаются через WebSocket
  - Сообщения приходят мгновенно обеим сторонам
  - При активном тикете сообщения НЕ идут к нейросети

- ✅ **Баннер "Оператор подключен"**:
  - Показывается ТОЛЬКО в чате с активным тикетом
  - Кеширование тикетов для каждого чата отдельно
  - Автоматически скрывается после решения тикета

- ✅ **Верстка панели оператора**:
  - Исправлена прокрутка сообщений
  - Текстовое поле и кнопка всегда видны внизу экрана

- ✅ **Логика отправки сообщений**:
  - Если есть активный тикет → через WebSocket, без нейросети
  - Если тикета нет → через API к нейросети
  - Автоматическая подгрузка истории тикета при открытии чата
