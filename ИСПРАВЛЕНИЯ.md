# Исправления системы поддержки

## ✅ Все исправленные проблемы

### 1. Ошибка sender_id
**Проблема:** `столбец messages.sender_id не существует`

**Решение:**
```bash
cd server
psql -d your_database -f add_sender_id.sql
# или
fix_sender_id.bat  # для Windows
```

### 2. Ошибка .unique() в SQLAlchemy
**Проблема:** `The unique() method must be invoked on this Result`

**Исправлено в:**
- ✅ `list_tickets()` - строка 80
- ✅ `get_ticket()` - строка 116

**Причина:** При использовании `joinedload()` с коллекциями (например, `messages`) необходимо вызывать `.unique()` перед `.scalars()` или `.scalar_one_or_none()`.

**Как избежать:** См. `server/SQLALCHEMY_TIPS.md`

### 3. Онлайн операторы
**Проблема:** Тикеты назначались всем операторам из БД, даже оффлайн

**Решение:**
- Добавлена функция `get_online_operators()` в `socket_events.py`
- Тикеты теперь назначаются только онлайн операторам (подключенным через WebSocket)
- Если операторов нет - пользователь видит сообщение о горячей линии

### 4. Таймауты OpenRouter
**Проблема:** `httpcore.ReadTimeout`, `httpx.ConnectError`

**Решение:**
- Увеличен таймаут до 120 секунд
- Добавлен retry механизм (3 попытки)
- Экспоненциальный backoff: 1, 2, 4 секунды
- Улучшена обработка ошибок сети

### 5. Кнопка "Позвать оператора"
**Проблема:** Кнопка не появлялась или работала некорректно

**Решение:**
- Упрощена логика отображения
- Кнопка появляется у последнего сообщения от бота
- Показывается только если нет активного тикета
